# vaultwarden-operator Helm Chart

A Kubernetes operator that syncs secrets from a [Vaultwarden](https://github.com/dani-garcia/vaultwarden)
(self-hosted Bitwarden) instance into native Kubernetes `Secret` objects using client-side
Bitwarden crypto.

## Installing

```sh
helm repo add vaultwarden-operator https://jordanruthe.github.io/vaultwarden-operator
helm repo update
```

### 1. Create the credentials Secret

The operator reads Vaultwarden credentials from a pre-existing Kubernetes Secret.
Create it before installing the chart:

```sh
kubectl create namespace vaultwarden-operator-system

kubectl create secret generic vaultwarden-operator-credentials \
  --namespace vaultwarden-operator-system \
  --from-literal=VAULTWARDEN_URL=https://vault.example.com \
  --from-literal=VAULTWARDEN_EMAIL=operator@example.com \
  --from-literal=VAULTWARDEN_PASSWORD=supersecret
```

For API-key auth (bypasses 2FA), also add `VAULTWARDEN_CLIENT_ID` and
`VAULTWARDEN_CLIENT_SECRET` to the Secret and wire them up via `extraEnv` in your values.

### 2. Install the chart

```sh
helm install vaultwarden-operator vaultwarden-operator/vaultwarden-operator \
  --namespace vaultwarden-operator-system \
  --create-namespace
```

## Upgrading

```sh
helm upgrade vaultwarden-operator vaultwarden-operator/vaultwarden-operator \
  --namespace vaultwarden-operator-system
```

The CRD is templated and gated by `installCRDs` (default `true`), so Helm will upgrade
it alongside the chart. The CRD is annotated with `helm.sh/resource-policy: keep` to
prevent accidental deletion of user resources on `helm uninstall`.

## Uninstalling

```sh
helm uninstall vaultwarden-operator --namespace vaultwarden-operator-system
```

**Note:** The CRD is retained after uninstall (resource-policy: keep). To remove it:

```sh
kubectl delete crd vaultwardensecrets.secrets.vaultwarden.io
```

## CRD sync note

`templates/crd.yaml` is a vendored copy of `config/crd/vaultwardensecret.yaml`, which
is generated by:

```sh
cargo run --bin crdgen > config/crd/vaultwardensecret.yaml
```

When the CRD schema changes, regenerate that file and copy it into `templates/crd.yaml`
(keep both in sync).
